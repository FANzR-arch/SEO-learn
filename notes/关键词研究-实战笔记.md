# 关键词研究实战笔记（脱敏版）

> 这是一次「从零做关键词研究」的完整实战复盘。已脱敏：隐去产品名，保留方法论。
> 标【补充】的段落是在原始记录之外补进来的方法论，用于查漏补缺。

## 0. 背景设定（脱敏）

- **本产品**：一款把普通 PDF 简历 / 作品集，变成「可互动、可对话」的在线个人主页工具。
- **三类目标用户**：创业者（让投资人快速理解项目）、设计师（让客户看到成果）、求职者（让招聘者更全面了解自己）。
- **本轮练习对象**：**求职者**——因为「产品 ↔ 需求」的连接最直接，最适合走一遍完整流程。选它不是因为它一定最优，而是它最适合练手。

---

## 1. 第一课：目标用户 ≠ SEO 关键词

想清楚「谁、想完成什么事」只是**起点**，不等于找到了关键词。中间还差两步验证：

1. 这些人遇到问题时，**会不会真的去搜索**？
2. 如果会，**他们会怎么搜**（用什么词）？

> **关键词研究的正确起点，是「用户想完成的事（jobs-to-be-done）」，不是「产品功能」。** 但起点之后必须用数据落地，否则只是自说自话。

**回答「你找的能准吗」：** 靠脑补列出来的词天然不准——它会系统性地①混入根本没人搜的词，②误判难度。所以脑暴列表只是**假设清单**，必须进 Ahrefs 逐个验证。**是的，一定要去 Ahrefs（或任何关键词工具）找。**

---

## 2. 关键词研究的完整流程

```
目标用户
   ↓
他遇到的具体问题
   ↓
他可能输入的搜索内容（脑暴 = 假设）
   ↓
用 Ahrefs 验证是否真有人搜（SV / KD / TP）
   ↓
判断本产品能否满足这个搜索需求（意图匹配）
   ↓
【补充】看真实 SERP，确认意图 + 竞争强度
   ↓
决定建什么页面（文章 / 落地页 / 对比页）
```

---

## 3. 搜索意图：同一句话，不同目的

用户搜一句话时，**真正想得到的结果**才是意图。表面相似的词，意图可能完全不同。

| 脑暴表达 | 用户真正想要 | 与本产品的关系 |
|---|---|---|
| 怎么做个人网站 | 找教程 / 建站工具 | 相关，但范围太宽 |
| 简历怎么更好地呈现给招聘者 | 找更好的简历展示方式 | **非常相关** |
| 简历要放什么内容 | 学习简历怎么写 | 有流量，但离产品较远 |

**【补充】搜索意图四分类**（比 Ahrefs 的 I/C/T 更细，是行业标准）：

| 类型 | 含义 | 例子 | 适合的页面 |
|---|---|---|---|
| 信息型 Informational | 想学东西、找答案 | how to write a resume | 教程、指南、案例 |
| 导航型 Navigational | 找特定网站 / 品牌 | linkedin login | 官网、品牌页（他人品牌抢不动） |
| 商业调查型 Commercial | 选之前做比较 | best resume builder | 对比页、评测、Alternative 页 |
| 交易型 Transactional | 准备行动 / 转化 | create online resume | 落地页、注册页、功能页 |

> Ahrefs 里的 **I / C / T** 就是它对这三类主意图的标注，可作为快速参考，但**最终以真实 SERP 为准**。

## 4. 三类关键词与内容漏斗

同一个用户，在不同阶段搜不同的词。把词按「离产品的远近」分三类：

| 类型 | 用户状态 | 例子 | 适合做 |
|---|---|---|---|
| **问题词** | 在学习 / 解决问题 | what to include in a resume；how to make a personal website | 教程、指南、案例文章 |
| **解决方案词** | 已知道大概要什么 | resume website；online resume；interactive resume | 场景落地页、功能页、模板页 |
| **产品比较词** | 准备选工具 | best resume website builder；alternatives to PDF resume | 对比页、工具推荐、Alternative 页 |

**核心规律：离产品越近，流量通常越少，但转化越好。**

**【补充】对应营销漏斗**（帮助理解「一个词该放在漏斗哪一层」）：

- 问题词 → **漏斗顶部 TOFU**（拉新、建立认知）→ 内容入口
- 解决方案词 → **漏斗中部 MOFU**（用户在考虑方案）→ 承接转化
- 产品比较词 → **漏斗底部 BOFU**（临门一脚）→ 直接转化

> 内容漏斗打法示例：用户搜「简历要写什么」→ 读你的简历结构指南 → 意识到 PDF 简历的局限 → 了解「互动简历」→ 创建自己的主页。**问题词不直接卖货，但它是把人引进漏斗的入口。**

## 5. 读懂 Ahrefs 数据（字段速查）

| 字段 | 现阶段理解为 |
|---|---|
| **SV** | 目标市场（如美国）月均搜索次数 |
| **GSV** | 全球月均搜索次数 |
| **KD** | 进前十的估算难度，0 最低、100 最高 |
| **TP**（Traffic Potential） | 当前排名第一的页面，从该主题**及相关词**能拿到的预估总流量 |
| **Parent Topic** | Ahrefs 判断这个词可能归属的**更大主题** |
| **I / C / T** | 信息 / 商业调查 / 交易 三种意图标注 |
| **CPC** | 广告主愿为一次点击付多少钱（间接反映商业价值） |

**【补充】三个容易踩的坑：**

1. **TP 常远大于 SV** —— 因为一个排名页面同时覆盖大量相关搜索，不只吃一个词。→ **做内容要瞄准「父主题」，而不是单个词。**
2. **KD 只看前十页面的引用域数量**，不考虑内容质量和站内优化。→ KD 只能初筛，不能单独决定「好不好排」。
3. **Parent Topic 是线索不是定论** —— 它根据当前第一名页面推断。若父主题和你的产品定位不符，是**警报**（见下方 `interactive resume` 案例）。

---

## 6. 三个反直觉原则

- **搜索量最高 ≠ 最好**：`online resume` SV 2K，但 KD 94，且意图很宽（多数人在找免费简历生成器，不是找互动主页）。
- **KD 最低 ≠ 最好**：`interactive resume` KD 0，但 Parent Topic 是 **animated resume（动画简历）**——搜的人可能想要「会动的简历」，不是「能对话的 AI 简历」。意图可能对不上。
- **真正要找的词**：**搜索需求明确 + 产品匹配 + 竞争可接受 + 你能做出正确的页面**，四个条件同时成立。

## 7. 本轮 12 个候选词的分类结论（脱敏）

| 类型 | 关键词 | 数据要点 | 判断 |
|---|---|---|---|
| 信息内容机会 | what to include in a resume | SV 4.1K / KD 22 / 信息型 | 值得做，但离产品远，靠内容漏斗引流 |
| 信息内容机会 | resume sections | SV 2.3K / KD 3 | 值得做，但**可能与上一个词合并**（见第 8 节） |
| 长期产品词 | resume website | SV 2.2K / **KD 92** / TP 144K | 产品匹配高，但竞争极强，需真正的功能页，非短期目标 |

## 8. 【补充】关键词同类相食（Cannibalization）

`what to include in a resume` 和 `resume sections` 意思高度重叠。若各写一篇：

- 文章 A：What to Include in a Resume
- 文章 B：Essential Resume Sections

两篇会**互相抢排名、分散权重**——这叫「关键词同类相食」。

**更优做法：用同一篇文章覆盖一组近义词**（what to include in a resume / resume sections / what should a resume include / essential resume sections）。是否合并，最终看这些词的**真实 SERP 是否高度重合**——如果 Google 给它们返回的是几乎一样的页面，就该合并。

---

## 9. 没有数据的词怎么办

`AI resume chatbot`、`shareable resume link` 等在 Ahrefs 里没数据，**不等于没人搜**，而是：

- Ahrefs 数据不足，**不能靠它单独撑起一个页面**；
- 但可以：作为文章里的**次级表达**、用于标题测试、通过 Google 自动补全继续扩展、上线后在 **Search Console** 里观察真实表现。

> 尤其是产品新概念词（如可对话简历），往往是「需求正在形成、工具还没数据」——适合内容测试，不适合把第一篇重头页面完全押上去。

## 10. 【实战】SERP 分析：以 `interactive resume` 为例

> **核心原则：搜索意图不能靠关键词字面猜，要看 Google 实际排出了什么。**
> 我们本以为 `interactive resume` = 「互动式个人介绍」，但查 SERP 后发现 Google 更偏向「动画简历 + 模板 + 工具 + 案例」——这也解释了 Parent Topic 为什么是 `animated resume`。

**第一步：给前十结果归类，看 Google 当前理解的需求**

| 结果类型 | 示例站点 | 搜索者可能想要 |
|---|---|---|
| 作品案例 | 个人作品站、Behance | 看别人做的互动简历长什么样 |
| 模板与工具 | Genially、高校指南页 | 直接上手做一个 |
| 简历产品 | 在线简历工具 | 在线创建普通简历 |
| 教程与讨论 | ThingLink、Reddit | 学怎么做、值不值得做 |

**结论**：搜索者没有统一目的，但主要集中在「带动画、点击交互、视觉展示的网页简历」，**不是**「可以让招聘者向 AI 提问的个人介绍」。→ 本产品（可对话式）和这个词相关，但属于**产品创新式解读**，不能假设搜这个词的人本来就在找 AI Chat。

**第二步：读懂「KD=0，但结果全是大站」这个矛盾**

- 现象：Reddit DR 95、Behance DR 94、Genially DR 85……站点整体都很强；
- 但很多**具体排名页面**的 UR、引用域数量却很低；
- 因为 **KD 主要看前十页面拿到多少引用域**，不看内容质量和站内优化。

> **KD=0 的正确理解**：不是「随便写就能第一」，而是「这个 SERP 里**部分具体页面的外链门槛不高**，值得进一步调查」。大站在榜 ≠ 大站的那一页真的强。

**第三步：找「机会信号」**

SERP 里同时出现：论坛讨论（Reddit）、作品搜索页（Behance）、图片包（Image Pack）、少数精准教程——**结果类型混杂**，通常说明 **Google 还没找到一个绝对占优的标准答案**。这就是新站的缝隙：一个内容足够完整的页面，有机会同时满足多种需求。

**第四步：从 SERP 反推「该做什么页面」**

不要只做一个纯产品页（"Create an AI Interactive Resume"）。更贴合当前搜索需求、又能自然引出本产品差异的结构是一个**综合页**：

```
什么是 interactive resume
→ 普通简历 / 网页简历 / 互动简历的区别
→ 互动简历案例
→ 常见互动方式（动画、点击、项目浏览）
→ 新方式：让招聘者直接提问（本产品的差异点）
→ 如何把现有简历变成互动个人页面
→ 创建入口
```

> 复述关键点：**每个重点关键词都要单独查 SERP**——意图不看字面，看 Google 实际排出什么。KD 是机器估的，SERP 是眼睛看的；两者冲突时，信 SERP。

**这个词最终怎么标记：**

| 项目 | 判断 |
|---|---|
| 搜索意图 | 案例 + 模板 + 工具 + 教程（混合） |
| 产品匹配度 | 中高 |
| 竞争数据 | 表面较低（需逐页核实外链门槛） |
| 主要风险 | 用户期待的是视觉/动画简历，不是 AI 对话 |
| 初步优先级 | **P1 测试词**（不定为核心产品词，也不放弃） |
| 适合页面 | 场景页 + 指南结合的综合页 |

## 11. 【实战】内容缺口（Content Gap）：拆 `interactive resume` 前三名

> 分析竞品页面，不能停在「它是什么类型」，要往前再走一步：判断它**满足了什么需求**、**还缺什么**。缺口 = 你的页面有理由超过现有结果的地方。

**三个真实页面逐个拆**（对应 SERP 里三种典型页型；Genially / ThingLink 已抓取确认，Leonardi 为公开知名案例）：

| 页面 | 类型 | 「互动」指什么 | 满足了什么 | 还缺什么 |
|---|---|---|---|---|
| 个人游戏化简历（Rleonardi） | 案例 | 游戏化滚动浏览 | 看优秀案例、找灵感、直接感受作者能力 | 无法复用、没模板/教程、只适合设计前端类；访客只能按设计好的路径看，**不能继续提问** |
| 模板 + builder（Genially） | 工具 | 动画、链接、多媒体 | 选模板、挑风格、用现成工具快速开始 | 核心是视觉模板、要自己填资料；互动=点击/动画，**无法按招聘者的问题动态回答** |
| 教程博客 + 产品导流（ThingLink） | 教程 | 热点、视频、链接、页面跳转 | 什么是/为什么/放什么/怎么设计/去哪拿模板/用什么工具 | 互动仍是「预先设计好的信息路径」，**用户能点什么取决于作者提前放了什么** |

**它们共同的缺口**：现有互动简历让页面更「丰富」，但仍要求招聘者**按作者提前设计好的路径**阅读，没解决「招聘者接下来想问什么」。

**好内容缺口的 4 个条件**（要同时满足）：
1. 现有搜索结果没有充分解决；
2. 用户确实可能需要；
3. 你（你的产品）有能力给出不同答案；
4. **不破坏原关键词的主要搜索意图**——这一条最关键。

> 反例（太浅）：「前三名都没提 AI，所以我写篇 AI 文章。」搜 `interactive resume` 的人主要想看 定义 / 案例 / 模板 / 制作方法 / 工具——你得**先把这些提供完整**，再加入你的新解法。

**从缺口反推页面结构**（综合页：先满足原意图，再引出差异）：
什么是 → 优秀案例 → 常见互动方式（动画 / 游戏 / 多媒体 / 作品集 / **对话式**）→ 该包含什么 → 普通互动简历的局限 → 本产品的新方式（可对话）→ 怎么把现有简历变成它 → Demo → 创建入口。

**这个词的记录卡**：意图 = 案例+模板+教程+工具；现有定义 = 动画/点击/多媒体/游戏化；内容缺口 = 无法按招聘者真实问题继续介绍候选人；本产品角度 = conversational（可对话）interactive resume；页面类型 = 指南+产品场景页；优先级 = **P1 测试**（还需和 `resume website`、`personal website for job seekers` 比较，未必是求职场景最好的词）。

**这一课真正要记住的**：分析竞品别停在「案例 / 工具 / 博客」，继续追问四连——用户为什么点进来？解决了什么？看完还缺什么？你能不能更好地补这个缺口？

## 12. 【实战·重要】内容缺口 ≠ 关键词

> 上一节找到的「缺口」是**产品 / 内容差异**，不一定是**用户会搜的词**。这两件事必须分开——很多人就栽在把「我想强调的差异点」当成「关键词」硬做。

**招聘者看完简历后，可能还想知道的（三类断层）：**
- 这个项目里他具体负责什么？
- 有没有做过类似行业？
- 为什么从 A 领域转向 B？
- 目前能接受什么合作方式？
- 作品背后的思考是什么？

传统互动简历只能让人**继续点击预设内容**；问不到答案，就得「找联系方式 → 加好友 / 发邮件 → 组织问题 → 等回复」。本产品的机会是在中间加一层：「看到简历 → 直接提问 → AI 用候选人给的资料回答 → 真值得再联系本人」——**缩短「产生兴趣」到「进一步了解」之间的断层**（不是替代邮件 / 加好友）。

**产品缺口一句话**（适合放进页面，但它不是关键词）：
> 现有互动简历大多是**可点击的**，不是**可对话的**。
> Most interactive resumes are clickable, but not conversational.

**关键区分：**

| | 是什么 | 有没有搜索量 |
|---|---|---|
| `interactive resume` | 用户已有的搜索需求（入口词） | **有**，真实流量入口 |
| 「可点击但不可对话」 | 你发现的产品 / 内容差异 | 没人会这么搜，**不是关键词** |

用户不会去搜 `clickable but not conversational resume`。所以页面要**先满足用户原本想看的**（案例 / 模板 / 教程），**再**介绍新方式。

**SEO 内容策略：进入 → 完成 → 展示**
> 从用户已有需求**进入** → **完成**用户原本的任务 → 再**展示**他还没意识到的新解法。
> （不是「把产品硬塞进一篇文章」。）

**页面定位模板**（可复用到任何「新概念产品」）：
- **关键词入口**：interactive resume
- **用户原本需要**：案例、模板、制作方式、工具
- **现有内容缺口**：看完简历后，无法马上就具体问题继续了解候选人
- **本产品差异**：让招聘者直接询问候选人的经历、项目和观点
- **页面承诺**：不仅让简历可浏览，也让它可以回答问题

**⏳ 待解决的判断题**（下一批展开）：当产品最准确的表达（`conversational resume` / `resume chatbot`）**没有搜索量**时，是**换词**，还是用**成熟关键词（`interactive resume`）承接新概念**？——需先看这组词的 Ahrefs 数据：`conversational resume`、`AI resume`、`AI resume website`、`resume chatbot`、`chat with resume`、`AI CV`、`interactive CV`、`resume AI assistant`。

## 13. 【补充】扩展的关键词候选（待验证假设）

回应「补充打词候选」。**以下全部是假设，必须进 Ahrefs 逐个验证 SV / KD / SERP，别直接用。** 按三类整理（求职者场景）：

**问题词（TOFU · 内容引流）**
- how to write a resume that stands out
- why is my resume not getting responses
- how to showcase projects on a resume
- resume vs portfolio difference
- how to make a resume more engaging

**解决方案词（MOFU · 承接转化）**
- interactive resume / interactive cv
- online resume / web resume / digital resume
- resume website / personal resume site / resume microsite
- shareable resume link
- one page personal profile

**产品比较词（BOFU · 临门一脚）**

> 三类人（创业者 / 设计师 / 求职者）都成立，但**求职者的词最具体、最好起步**；设计师方向可重点验证 `interactive portfolio` 与 `portfolio website` 的意图差异；创业者方向意图最分散（融资材料 / 公司官网 / Pitch Deck 混在一起），需要 SERP 逐词拆。

## 一句话总结

> **关键词研究不是「找词」，而是「验证假设」**：从用户要完成的事出发 → 脑暴假设 → 用数据（SV/KD/TP）筛 → 用真实 SERP 定意图和竞争 → 再决定建什么页。搜索量最高、KD 最低都不是目标，「意图明确 + 产品匹配 + 竞争可接受 + 你能做对页面」才是。
