第八课 · 阶段二 关键词与搜索意图

把词，分配给页面

关键词研究的最后一步，是把一堆词变成一张网站规划图：哪些词共用一页、哪些必须分开、每页只答一个问题。这就是关键词映射，也是阶段二的收尾。

1 关键词聚类

目的相同的词，共用一个页面

关键词聚类的意思是：表达不同、但搜索目的相同的词，应该由同一个页面承接。

比如 resume chatbot 和 chatbot resume，只是字序不同，用户想找的其实是同一种东西。如果分别建两个页面：

  • /resume-chatbot 和 /chatbot-resume 内容会高度重复；
  • 两个页面还会互相抢排名——这就是「关键词内耗」（也叫同类相食）。
快速判断两个词能不能共用一页
分别搜这两个词，各看前十名结果：≥5 个 URL 相同 → 通常可以由同一页承接；只有 1–2 个相同 → 很可能是两种不同需求。顺序不同没关系，重点看 URL 是否重复。
2 相关性看「任务」

别看词里有哪些字，看用户在完成什么

最容易犯的错，是看到共同的单词就以为相关。下面三个词都含 chatbot 和 resume，但用户任务完全不同。假设你在做一个「把简历变成可提问 AI」的产品，试着把每个词归到对的页面，或排除掉：

看出来了吗？单词几乎一样，需求可以完全相反。判断相关性的唯一标准，是「搜索者正在完成什么任务」。

3 合并还是拆分

三个问题，决定一页还是两页

手里的词该并成一页、还是拆成几页？不看词长得像不像，看这三个问题的答案是否相同：

1用户任务
他来干嘛

看案例？还是创建产品？任务不同就该分开。

2理想页面
该给什么

指南集合？还是产品页 + Demo？页面类型不同就该分开。

3主 CTA
让他做什么

了解学习？还是上传创建？行动目标不同就该分开。

三个都相同 → 合并；有一个不同 → 拆分。 拆开后，两个页面还应该互相链接，让用户和搜索引擎都明白它们的主题关系。

4 关键词地图

研究的产出物：一页答一个问题

把上面的判断落到纸面，就是一张关键词地图——每个页面一行，钉死它的主词、意图、页面类型和 CTA：

URL主关键词页面类型主要目的主 CTA
/interactive-resumeinteractive resume指南 + 案例拿更宽泛的互动简历流量轻（先看 / 体验）
/resume-chatbotresume chatbot产品 + Demo拿高匹配的产品流量直接（上传创建）

注意最后一列：页面目标 ≠ CTA。目标是「用户理解什么」，CTA 是「用户接下来做什么」。而且 CTA 的强弱要跟着意图走——

意图越偏学习，CTA 越轻；越接近产品，CTA 越直接
搜 interactive resume 的人多半在看案例、找灵感，还没准备创建 → 主 CTA 先让他「看一份可对话简历怎么运作」；搜 resume chatbot 的人已经离产品很近 → 可以直接「上传简历，创建你的 Chatbot」。

你刚刚走完的这条链路，就是关键词研究变成网站规划的全过程：
关键词 → 搜索意图 → 页面类型 → 页面目标 → CTA。

5 随堂自测

点一点、填一填，检查你会了没

本课要点
  • 关键词聚类：目的相同的词共用一页；字序不同不代表要分页；
  • 相关性看用户任务，不看词里有哪些字；
  • 合并 / 拆分看三问：用户任务 / 理想页面 / 主 CTA，都相同才合并；
  • 产出物是关键词地图：一页一个主词、一个意图、一个 CTA；意图越近产品，CTA 越直接。
